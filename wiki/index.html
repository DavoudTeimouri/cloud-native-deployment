Architecture Overview

This deployment uses a transparent reverse proxy architecture. All clients use canonical public URLs (like archive.ubuntu.com, docker.io) and the reverse proxy handles routing internally.

Kubernetes

What is it?

Kubernetes (K8s) is a container orchestration platform that automates deployment, scaling, and management of containerized applications. Originally developed by Google, now maintained by CNCF.

Core Concepts

ConceptDescription
PodSmallest deployable unit — one or more containers sharing network/storage
DeploymentDeclarative updates for pods with rollout/rollback
StatefulSetLike Deployment but for stateful apps (stable network IDs)
DaemonSetOne pod per node (logging, monitoring agents)
ServiceStable network endpoint for a set of pods
IngressHTTP/HTTPS routing to services
NamespaceVirtual cluster for resource isolation
PersistentVolumeCluster-wide storage resource
StorageClassDefines storage types (SSD, HDD, etc.)

Why KubeSpray?

  • Bare-metal/VM focused — works on any infrastructure
  • Highly configurable — every component customizable
  • Supports any CNI (Calico, Cilium, Flannel)
  • Air-gap friendly — can deploy without internet
  • CNCF certified — passes conformance tests
Tip: For small clusters (<5 nodes), consider K3s — a lightweight, single-binary Kubernetes distribution.

Ceph

What is it?

Ceph is a distributed storage platform providing object, block, and file storage from a single cluster. Designed for scalability, reliability, and no single point of failure.

Core Concepts

ConceptDescription
OSDObject Storage Daemon — stores data, handles replication
MONMonitor — maintains cluster map and health
MDSMetadata Server — for CephFS filesystem
PGPlacement Group — how data is distributed across OSDs
RBDRADOS Block Device — block storage for VMs/containers
CephFSPOSIX-compliant filesystem
RGWRADOS Gateway — S3-compatible object storage
CRUSHAlgorithm that determines data placement

Why Ceph?

  • Unified storage: Block (RBD), file (CephFS), object (RGW) in one cluster
  • Self-healing: Automatic data rebalancing when OSD fails
  • Scalability: Petabytes+ with no single point of failure
  • Kubernetes integration: CSI driver for persistent volumes
Tip: For object-only storage, MinIO is a lighter alternative. For K8s-only block storage, consider Longhorn.

Networking

Calico (CNI)

Calico is a container networking plugin providing networking and network policy enforcement. It routes pod traffic between nodes using BGP.

MetalLB (Load Balancer)

MetalLB provides LoadBalancer type services for bare-metal Kubernetes clusters. It assigns IPs from a configured pool and announces them via ARP (L2) or BGP.

NGINX Ingress

NGINX Ingress Controller manages external access to services, providing HTTP/HTTPS routing, TLS termination, and rate limiting.

CoreDNS

CoreDNS is the default DNS server in Kubernetes. It provides service discovery and name resolution within the cluster.

Monitoring

Prometheus

Prometheus collects metrics from instrumented targets, stores time-series data, and provides PromQL for querying. It pulls metrics (doesn't need agents to push).

Grafana

Grafana connects to Prometheus (and other datasources) to create visual dashboards for monitoring.

Loki

Loki is a horizontally scalable log aggregation system. It indexes labels (like Prometheus) not full text, making it cheap to operate.

Why Prometheus + Grafana + Loki?

  • Pull-based: Prometheus scrapes metrics — no need to push
  • Dimensional data: Metrics with labels (namespace, pod, node)
  • Service discovery: Auto-discovers K8s pods and services
  • Air-gap friendly: All self-hosted, no SaaS dependency

Security

Pod Security Standards

Kubernetes Pod Security Standards (PSS) enforce security policies at the namespace level. Three levels:

  • Privileged — Unrestricted (for system components)
  • Baseline — Minimally restrictive (prevents known privilege escalations)
  • Restricted — Heavily restricted (security best practices)

Network Policies

NetworkPolicies control traffic flow between pods. By default, all pods can communicate. Apply deny-all policies and allow specific traffic.

Automates TLS certificate management from various sources (Let's Encrypt, Vault, self-signed). Essential for enabling HTTPS everywhere.

Vault

HashiCorp Vault provides secrets management, encryption as a service, and dynamic secrets generation. Replaces native K8s Secrets for production.

Automation

Ansible

Ansible is an agentless automation tool using YAML playbooks. Connects via SSH — no software to install on targets. Used for OS preparation and KubeSpray deployment.

ArgoCD (GitOps)

ArgoCD is a GitOps continuous delivery tool. It watches a Git repository and automatically syncs the desired state to the cluster.

AWX (Ansible Web UI)

AWX is the open-source version of Ansible Tower. Provides a web UI, job scheduling, RBAC, and REST API for Ansible playbooks.

Velero (Backup)

Velero provides backup and disaster recovery for Kubernetes. Backs up cluster resources and persistent volumes, can restore to same or different cluster.

Reverse Proxy Architecture

How It Works

The reverse proxy sits between clients and backend services. It intercepts requests to public domains and routes them to the appropriate internal service.

DNS Configuration

All public repository domains resolve to the proxy IP:

archive.ubuntu.com    → 10.0.0.10 (proxy)
registry-1.docker.io  → 10.0.0.10 (proxy)
quay.io               → 10.0.0.10 (proxy)
registry.k8s.io      → 10.0.0.10 (proxy)

SSL Options

OptionHowClient Changes
Custom CAGenerate your own CA, install on all clientsInstall CA cert once

>
Important: For Docker registry proxying, you need to install your internal CA certificate on ALL client machines. This is a one-time operation.

Air-Gap Consideration

In air-gap mode, the proxy cannot reach the internet. All data must be pre-staged into Nexus (packages) and Harbor (container images) before disconnecting.
